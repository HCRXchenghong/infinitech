# 实施清单

这份清单只记录当前“管理端盘活 + 平台统一收款自研分账 + 全端真实支付/充值/提现 + 骑手保证金 + 银行卡提现方案”的真实落地状态。做完的内容会从待完成里移除，未完成的内容会继续往下收。

## 已完成

### 管理端与导航
- 管理端导航已经升级为“分组 -> 小节 -> 菜单项”的三层结构，保留了现有骨架，没有推倒重做。
- 财务组已新增“支付中心”，旧“付款说明管理 / PaymentInfo”入口已下线。
- 原空白占位页已盘活为“联调工作台”。
- 顶部面包屑和导航结构已同步细化。

### 支付中心
- 已支持分端分场景支付矩阵配置。
- 已支持提现手续费规则配置。
- 已支持分账对象、分账规则和规则预览。
- 已支持骑手保证金策略配置。
- 已支持银行卡提现 `sidecar / provider` 配置。
- 已支持支付回调与出款日志查询：
  - 可按渠道、事件、处理状态、验签状态筛选。
  - 可按关联交易号、第三方单号精确筛选。
  - 可查看关联交易、关联提现单、请求头、请求体、响应体详情。
- 已支持“订单分账查询”：
  - 可按订单 ID / UID / TSID / 日订单号查询清分快照。
  - 可查看分账分录、订单概览与快照 JSON。
- 已增加配置保存时的后端校验：
  - 用户端不允许银行卡提现。
  - 小程序不允许支付宝支付/充值。
  - `rider_deposit` 仅允许骑手 App 的微信/支付宝。
  - 已启用提现渠道必须存在手续费规则。
  - 手续费区间不允许重叠。

### 后端账务主链
- 订单支付、充值、保证金支付已改成第三方渠道状态流，不再“提交即成功”。
- 提现已改成统一状态机：
  - `pending_review`
  - `pending_transfer`
  - `transferring`
  - `success`
  - `failed`
  - `rejected`
- 订单退款已改成：
  - `refund_pending -> callback success -> refunded`
- 提现状态查询已支持主动查单补偿。
- 微信 / 支付宝提现自动对账 worker 已接入服务启动链。
- 支付回调已补齐幂等和乱序保护。
- 后台提现处理已支持：
  - 通过
  - 驳回
  - 发起打款
  - 同步状态
  - 失败后重试打款
  - 补记成功
  - 补记失败

### 分账系统
- 已落地结算对象、规则集、规则步骤、订单清分快照、结算分录模型。
- 已支持四种规则：
  - `percent_of_gross`
  - `tiered_by_order_amount`
  - `fixed_amount`
  - `remainder`
- 已接入主链：
  - 支付成功生成待分账快照
  - 订单完成正式入账
  - 退款执行分账冲销

### 骑手保证金
- 已支持 50 元保证金策略。
- 已接入“未缴保证金不可接单”的基础门禁。
- 已接入“7 天未接单且无进行中订单可提现”的基础判断。
- 保证金提现已并入统一提现链路。
- 保证金自动提现时已支持自动发起打款。

### 银行卡提现
- 银行卡提现仅开放给骑手端和商户端。
- 银行卡提现到账时效已固定展示为“24 小时 - 48 小时”。
- 银行卡提现申请时会先冻结金额，并纳入手续费结算。
- 管理端已支持银行卡提现人工处理闭环：
  - 待处理列表
  - 人工打款完成
  - 驳回并返还冻结金额
  - 上传凭证
  - 保存详细出款来源信息
- 打款凭证和出款来源仅管理端可见。
- 银行卡回调验签路径和 sidecar 健康检查已接入。

### 客户端资金体验
- 用户小程序、用户 App、骑手端、商户端的钱包充值 / 提现已接入状态轮询。
- 订单确认页已按后台支付能力动态显示支付方式。
- 三端账单详情已接入单笔交易状态查询。
- 后端已新增增强版提现选项输出，会按渠道返回：
  - `arrivalText`
  - `accountPlaceholder`
  - `accountHint`
  - `namePlaceholder`
  - `requiresName`
  - `requiresBankName`
  - `requiresBankBranch`
  - `bankNamePlaceholder`
  - `bankBranchPlaceholder`
  - `reviewNotice`
- 后端已新增银行卡提现字段校验：
  - 必须填写收款人姓名
  - 必须填写开户银行
  - 必须填写开户支行
- 用户小程序提现页已重写，去掉重复方法并接入动态字段提示。
- 用户 App 提现页已重写，去掉重复方法并接入动态字段提示。
- 骑手端提现页已重写，银行卡提现会按后端规则收集持卡人、开户行、开户支行。
- 商户端钱包页已重写，提现流程不再只收银行卡号，现已完整收集：
  - 收款账户
  - 收款人姓名
  - 开户银行
  - 开户支行

## 2026-04-02 本轮续推进
- 提现失败后的自动重试元数据已接入提现状态接口，管理端支付中心和联调工作台都能直接看到“待自动重试”状态。
- 提现处理队列已新增“自动重试待执行”统计，便于运营判断哪些失败提现会由系统继续补偿。
- 提现执行阶段如果第三方网关立即返回失败，系统会立刻回滚冻结金额、把提现单改回失败，并写入下一次自动重试计划。

  - 提现执行即时失败后的余额回滚与重试计划

- 新增“回调日志”能力：
  - Go 后端新增支付回调列表和详情接口
  - BFF 新增 `/api/admin/wallet/payment-callbacks` 及详情代理
  - 支付中心新增“回调日志”页签与详情抽屉
  - 回归测试覆盖回调日志筛选、详情解析和关联提现单信息
- 新增“订单分账查询”能力：
  - 支付中心可按订单号查询清分快照和结算分录
  - 可查看订单概览、快照状态、分录列表和快照 JSON
  - 回调日志筛选已补充“关联交易号 / 第三方单号”精确检索
  - 已新增回归测试覆盖“有快照 / 无快照”两种订单分账查询场景
