# 实施清单

说明：
- 这份清单只记录当前这轮“支付中心 + 自研分账 + 充值提现 + 骑手保证金 + 银行卡提现 + 管理端盘活”的真实落地状态。
- 已经写入代码并通过本地编译或测试验证的，放到“已完成”。
- 仍然只是骨架、占位、半成品，或者还没做完真机联调和真实资金验收的，放到“待完成”。

## 已完成

### 1. 管理端导航与信息架构
- 管理端侧边栏已经升级为“分组 -> 小节 -> 菜单项”的三层结构。
- 保留了原有主骨架，没有推倒重来。
- 所有现有后台主路由都已经重新归位，不再只是一层平铺。
- 已补上之前有路由但没在导航中露出的“骑手等级”入口。
- 顶部路径显示已跟随导航结构升级，能显示更清晰的层级路径。

### 2. 管理端主链盘活
- 旧的“付款说明管理 / PaymentInfo”入口已下线。
- 财务组内已新增“支付中心”，并承接原先散落的支付配置。
- 原空白占位页已经盘活为“联调工作台”。
- 联调工作台可查看服务健康、支付网关就绪度、保证金与提现待处理概览。
- 支付中心已纳入银行卡出款就绪度、微信/支付宝出款回调与侧车配置。

### 3. 支付中心配置能力
- 已支持分端分场景支付矩阵配置。
- 已支持提现手续费规则配置。
- 已支持分账对象、分账规则、规则预览配置。
- 已支持骑手保证金、银行卡提现时效配置。
- 已支持银行卡出款侧车配置：
  - `sidecar_url`
  - `provider_url`
  - `merchant_id`
  - `api_key`
  - `notify_url`
  - `allow_stub`
- 已补齐支付中心保存时的后端校验：
  - 小程序不能配置银行卡。
  - 小程序不能配置支付宝支付/充值，只能配置支付宝提现。
  - 用户端不能配置银行卡提现。
  - `rider_deposit` 只能是骑手 App 的微信或支付宝。
  - 已启用提现渠道必须绑定手续费规则。
  - 手续费区间不能重叠。

### 4. 后端账务与状态机
- 订单支付、钱包充值、骑手保证金创建后，第三方渠道不再“提交即成功”。
- 退款已改成 `refund_pending -> callback success -> refunded` 的状态流。
- 提现已改成 `pending_review -> pending_transfer -> transferring -> success/failed/rejected` 的状态流。
- 提现状态查询已接入主动查单补偿：
  - 微信提现会主动查官方转账状态。
  - 支付宝提现会主动查侧车转账状态。
  - 查到成功或失败会直接回写提现单和钱包流水。
- 已补齐钱包状态查询接口：
  - `/api/wallet/transactions/:transactionId`
  - `/api/wallet/recharge/status`
  - `/api/wallet/withdraw/status`
- 后台提现处理已支持：
  - 通过
  - 驳回
  - 发起打款
  - 同步网关状态
  - 失败后重试打款
  - 补记成功
  - 补记失败
- 微信/支付宝提现自动对账 worker 已接入服务启动链，并在 `/ready` 暴露运行状态。
- 支付回调已补齐幂等保护、乱序保护，避免重复落库和成功状态被失败回调打回。

### 5. 分账基础模型
- 已落结算对象、规则集、规则步骤、订单清分快照、结算分录模型。
- 已支持规则预览：
  - 按订单总额比例
  - 固定金额
  - 骑手阶梯
  - 剩余金额
- 已接入“支付成功生成待分账快照 -> 订单完成正式入账 -> 退款冲销”的基础账务链。

### 6. 骑手保证金
- 已有骑手保证金策略、记录模型与状态查询。
- 已接入“未缴纳 50 元保证金不可接单”的基础门禁。
- 已接入“7 天未接单且无进行中订单可提现”的基础判断。
- 保证金提现已并入统一提现状态流。
- 保证金自动提现时已支持自动发起打款执行。

### 7. 银行卡提现人工处理闭环
- 银行卡提现申请时，金额会先冻结，手续费也会参与结算。
- 银行卡提现仅开放给商户端和骑手端。
- 管理端已新增银行卡处理区，展示待处理银行卡提现。
- 管理员人工完成银行卡打款时，必须填写并保存：
  - 打款凭证
  - 出款银行
  - 出款支行
  - 出款卡号
  - 出款账户名
  - 打款流水号/参考号
- 银行卡提现支持驳回：
  - 必须填写驳回原因
  - 冻结金额会返还
- 打款凭证和出款来源信息仅管理端可见，不会返回给客户端状态查询接口。

### 8. 多端前端基础闭环
- 用户端、小程序端、骑手端、商户端的钱包充值/提现页面已接入状态轮询。
- 订单确认页已按后端支付能力动态显示支付方式。
- 三端账单详情已接入单笔交易状态查询。
- 商户端已补齐充值/提现状态轮询与银行卡提现到账时效提示。

### 9. 真实 SDK 主链推进
- Go 后端已接入微信官方 `wechatpay-go` 依赖。
- 支付宝侧车已接入官方 `alipay-sdk` 依赖。
- 微信官方 Go SDK 已接入：
  - App 支付下单
  - 小程序 JSAPI 下单
  - 支付回调验签与解密
  - 退款创建
  - 商家转账 `transferbatch` 提现提交
- 支付宝官方侧车已接入：
  - App 支付下单
  - 退款请求
  - 回调验签
  - 提现转账请求提交
- 支付/提现回调已区分内部单号与第三方流水号，避免对账污染。

- `go build ./cmd`

- `node --check backend/alipay-sidecar/server.js`
- `node --check backend/bank-payout-sidecar/server.js`
