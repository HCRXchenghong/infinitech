# Infinitech

统一生活服务平台单仓代码库，目标是做成可持续演进的一站式平台，覆盖用户、商家、骑手、管理后台、实时消息、运营配置、支付/财务、地图、通知，以及可选的 AI 员工能力。

本仓库现在只保留这一份文档。之前分散在各目录里的部署说明、整改记录、配置说明、阶段总结和设计蓝图，已经统一收口到这里。

## 1. 项目定位

平台当前不是“演示项目”，而是在按大平台标准持续整治：

- 目标不是口头承诺“单机 10 万在线”
- 目标是把现有系统逐步改造成可水平扩展、可压测、可回滚、可观测、可持续维护的生产架构
- 优先整治主链路：认证、下单、消息、地址、通知、配置、后台运营、支付与财务基础能力
- 所有修复都按批次推进，每一批都要求验证通过后再提交并推送


## 5. 生产基线要求

当前已经明确的硬要求：

- 生产数据库采用 `PostgreSQL`
- 生产缓存采用 `Redis`
- `SQLite` 只允许本地开发、临时调试、兼容性场景，不再作为生产主链路前提
- 主链路消息事实源以服务端数据库为准
- 实时 Socket 只负责实时交互，不允许继续承担长期业务事实源角色
- 稳定文案不再盲目配置化，只有真正需要运营配置的数据才放后台

## 6. 端口规范

当前约定口径：

- `1029`：Go API 主服务端口
- `25500`：BFF 端口
- `9898`：Socket Server
- `9899`：HEIC 转换服务
- `2550`：Redis 本地开发默认端口
- `5432`：PostgreSQL 常规端口
- `3306`：MySQL 历史兼容端口，不再作为主计划路径

说明：

- 未经明确批准，不改核心端口口径
- 临时测试端口只允许短期使用，结束后必须恢复

## 7. 配置原则

### 通用原则

- 本地默认只允许回环地址，如 `127.0.0.1`
- Android 模拟器允许使用 `10.0.2.2`
- 不允许再把开发机局域网 IP 写回源码默认值
- 外链、邀请、领券、下载、地图、微信登录、短信、支付等都必须来自环境变量或后台配置
- 不通过修改 `dist`、`unpackage`、日志、数据库文件来“修配置”

### 重点配置项

- API 基础地址
- Socket 地址
- PostgreSQL 连接
- Redis 连接
- JWT / Admin JWT
- Aliyun SMS
- 微信登录
- 地图服务 / 天地图 Key
- 邀请链接与领券链接
- 应用下载地址

## 8. 本地开发启动

### 8.1 基础设施

```bash
cd backend/docker
docker compose up -d
```

### 8.2 Go API

```bash
cd backend/go
go run cmd/main.go
```

### 8.3 BFF

```bash
cd backend/bff
npm install
npm start
```

### 8.4 Socket Server

```bash
cd socket-server
npm install
node index.js
```

### 8.5 管理后台

```bash
cd admin-vue
npm install
npm run dev
```

### 8.6 uni-app 各端

按各自目录使用 HBuilderX / 对应运行方式调试：

- `user-vue`
- `app-mobile`
- `merchant-app`
- `rider-app`

## 12. 目前已经完成的整治摘要

这里不再逐条列 40+ 个历史 Markdown，而按能力面汇总。

### 12.1 安全与鉴权

- 修复了 socket token 签发过宽、身份可伪造、消息发送者可伪造的问题
- 修复了客服房间越权进入、越权加载聊天记录的问题
- 管理端鉴权不再依赖探活业务接口判断 token 是否有效
- BFF / Go / Socket 的关键密钥来源逐步统一

### 12.2 配置与部署基线

- 清掉了大量硬编码内网 IP
- 建立了更明确的运行时配置边界
- 强化了服务地址、邀请地址、领券地址、地图地址、HEIC 服务地址的可配置化
- 明确了 PostgreSQL + Redis 的生产基线

### 12.3 用户主链路

- 登录、注册、短信、图形验证码、微信登录已不是演示态
- 个人资料、改手机号、地址簿、下单地址绑定都已转向服务端真实数据
- 首页已改成服务端数据优先

### 12.4 商家与骑手主链路

- 骑手任务异常上报、历史订单、客服路径、部分任务详情已脱离假打通
- 商家聊天、订单联系入口已开始向服务端消息会话统一

### 12.5 实时消息

- 用户端与 App 端消息列表、聊天页已改为服务端会话优先
- 商家端聊天页已改为服务端历史优先
- 后台客服 / 监控工作台已开始改用服务端会话与历史接口
- Socket 现在会把消息与已读状态同步回 Go 服务

### 12.6 工程质量

- 修掉了大量活跃页面乱码
- 清掉了大量假数据、假成功提示和死逻辑
- 已把多轮整改记录按批次推进并持续验证、提交、推送
